## Run the demo

The quickest way to see OpenGraphs in action is with the demo training script:

```bash
og run demo_train.py --auto autonomous
```

This command:
- Starts a simulated training run
- Tracks metrics (loss, accuracy, learning rate)
- Collects system stats (GPU/CPU usage)
- Displays live graphs in your terminal

<Tip>
The demo script intentionally uses suboptimal hyperparameters to demonstrate how OpenGraphs helps you identify and fix training issues.
</Tip>

## Understanding the output

While the demo runs, you'll see:

### Console logs

```text

step    2 | loss=2.4156 | acc=0.1456 | base_lr=0.0010 | eff_lr=0.0004
step    3 | loss=2.3421 | acc=0.1689 | base_lr=0.0010 | eff_lr=0.0006
...
```

### Live TUI (if enabled)

The terminal UI shows:
- **Graphs tab**: Real-time metric plots
- **Logs tab**: Training output
- **Chat tab**: AI agent interface (with `--auto`)

## Explore the demo script

The demo script (`demo_train.py`) simulates a training loop with:

<CodeGroup>

```python Metric reporting

def report(metric, value, step):
    try:
        send_metric(metric, value, step=step)
    except Exception:
        pass

report("train/loss", loss, step)
report("train/accuracy", acc, step)
report("train/lr_effective", eff_lr, step)
```

## View metrics

After the run completes, you can query metrics using the CLI:

<Steps>

<Step title="List projects">
```bash
og list projects
```

Shows all projects in your runs directory.
</Step>

<Step title="List runs">
```bash
og list runs --project myproject
```

Shows all runs for a specific project.
</Step>

<Step title="View specific metric">
```bash
og get metric --project myproject --run <run-id> --metric train/loss
```

Displays all values for a specific metric.
</Step>

<Step title="Compare runs">
```bash
og compare --runs run1,run2 --metric train/accuracy
```

Compares the same metric across multiple runs.
</Step>

</Steps>

## Customize the run

You can customize the demo with environment variables:

<CodeGroup>

```bash Adjust learning rate
DEMO_LR=0.005 og run demo_train.py
```

```bash Change batch size
DEMO_BATCH=64 og run demo_train.py
```

```bash Both together
DEMO_LR=0.002 DEMO_BATCH=128 og run demo_train.py
```

</CodeGroup>

## Runtime backends

OpenGraphs supports different execution environments:

### Local runtime (default)

```bash
og run demo_train.py --runtime local
```

Runs the script on your current machine.

### Modal runtime

```bash
og run demo_train.py --runtime modal
```

<Note>
Modal runtime is currently a scaffold that executes locally while the remote adapter is finalized.
</Note>

## Specify metrics to graph

Control which metrics appear in the live graph view:

```bash
og run demo_train.py --graph '{"metrics":["loss","reward"],"sys":["gpu","vram"]}'
```

This displays:
- Training metrics: `loss` and `reward`
- System metrics: `gpu` and `vram`

## Customize metric labels

Make graphs more readable with custom labels:

<CodeGroup>

```bash JSON format
og run demo_train.py \
  --graph-labels '{"train/loss":"Loss","train/accuracy":"Accuracy"}'
```

```bash CSV format
og run demo_train.py \
  --graph-labels "train/loss=Loss,train/accuracy=Accuracy"
```

```bash Environment variable
export OG_GRAPH_LABELS='{"train/loss":"Loss","train/accuracy":"Accuracy"}'
og run demo_train.py
```

</CodeGroup>

## Use the AI agent

The `--auto autonomous` flag enables an AI agent that:
- Watches your training metrics in real-time
- Identifies issues (divergence, instability, suboptimal convergence)
- Suggests hyperparameter fixes
- Can automatically refactor your code (with permission)

<Warning>
The AI agent requires an OpenAI API key. Set it before running:
```bash
export OPENAI_API_KEY="your-key-here"
```
</Warning>

To use the agent:

```bash
export OPENAI_API_KEY="sk-..."
og run demo_train.py --auto autonomous
```

The agent will analyze the demo's intentionally bad hyperparameters and suggest improvements like:
- Reducing learning rate from 0.008 to 0.001
- Increasing warmup steps from 3 to 20
- Adjusting batch size for smoother updates

## TensorBoard integration

OpenGraphs can also read TensorBoard event files:

```bash
# Your code writes to TensorBoard
from torch.utils.tensorboard import SummaryWriter
writer = SummaryWriter(log_dir="runs/experiment")
writer.add_scalar("train/loss", loss, step)

# Point OpenGraphs to the same directory
og --path runs/experiment
```

## Common CLI commands

Here are the most useful commands for day-to-day usage:

```bash
# Run training with live graphs
og run train.py --runtime local --auto autonomous

# Tail logs from a specific run
og tail <run-id>

# Resume from checkpoint
og resume <run-id> --checkpoint latest

# Search for metrics
og search metrics --query loss

# Get all commands in JSON format
og list projects --json
```

Every command supports `--json` output for programmatic usage.
